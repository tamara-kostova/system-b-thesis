# Phase 0 — Data Setup

## What's loaded

1,119 synthetic patients generated by Synthea and loaded via `synthea_to_omop.py`:

| Table | Rows |
|---|---|
| person | 1,119 |
| condition_occurrence | 39,279 |
| drug_exposure | 48,451 |
| measurement | 309,418 |

## To regenerate

```bash
# Generate fresh Synthea patients (requires Java, synthea JAR in synthea/)
java -jar synthea/synthea-with-dependencies.jar -p 1000 \
  --exporter.csv.export=true --exporter.fhir.export=false

# Load into Postgres
python sql/synthea_to_omop.py --csv synthea/output/csv/
```

## Notes

- Postgres runs on **port 5433**
- R + Eunomia path was abandoned due to rJava compilation issues
- ETL-Synthea R package was also skipped for the same reason
- `sql/etl.py` handles SQLite/CSV but is superseded by `synthea_to_omop.py` for this setup
- SNOMED codes > 2,147,483,647 (newer COVID-era codes) are stored as concept_id = 0 (unmapped)
- Source values > 50 chars are truncated to fit OMOP VARCHAR(50) columns
